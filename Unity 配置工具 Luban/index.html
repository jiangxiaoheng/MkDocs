配置工具 Luban

参考:快速开始 | Focus Creative Games (focus-creative-games.github.io)

安装

  1. 环境搭建, 安装 dotnet sdk 6.0

  2. 下载 luban_examples项目 项目中包含测试配置、最新的 Luban Release 以及大量的示例项目。为方便起见，后续提及到的文件，默认都指这个项目中的文件

配置和代码生成

  1. 创建 Unity 工程, 复制 luban_examples\Projects\Csharp_Unity_json\Assets\LubanLib 目录;

同时在 Unity 的 PlayerSettings / OtherSettings 里开启 Allow 'unsafe' Code 选项 2. 从 luban_example 复制工具库 Tools\Luban.ClientServer 和 MiniTemplate 文件夹

目录及文件说明如下

luban_examples
    ├─Luban.ClientServer               # Luban 工具前后端一体
    └─MiniTemplate
        │    .cache.meta               # luban 缓存文件
        │    check.bat                 # bat 检查配置数据合法性
        │    check.sh                  # sh 检查配置数据合法性
        │    gen_code_json.bat         # bat 生成脚本
        │    gen_code_json.sh          # sh 生成脚本
        │
        ├─Datas                        # 数据源目录
        │      item.xlsx               # excel 数据源文件
        │      __beans__.xlsx          # bean 定义文件，可以使用xml定义代替
        │      __enums__.xlsx          # enum 定义文件，可以使用xml定义代替
        │      __tables__.xlsx         # table 定义文件，可以使用xml定义代替
        │
        ├─Defines
        │       __root__.xml           # luban 根文件
        │
        ├─Gen                          # 生成的代码目录    ,建议配置到 Unity/Assets/Gen 目录下
        └─output_json                  # 生成的json文件目录,建议配置到 Unity/output_json 目录下

生成命令

  • %Luban.ClientServer.dll% 指向 luban_examples/Tools/Luban.ClientServer/Luban.ClientServer.dll
  • --define_file 参数为 MyConfigs/Defines/root.xml 的路径
  • --input_data_dir 参数为 MyConfigs/Datas 的路径
  • --output_code_dir 参数为生成的代码文件存放的路径。 建议建议指向 unity的 Assets 目录下的某级子目录
  • --output_data_dir 参数为生成的数据文件的存放路径。 建议指向Assets同级目录。

gen_code_json.sh

#!/bin/zsh
WORKSPACE=.. # 工作区目录; Unity工程根目录

GEN_CLIENT=${WORKSPACE}/Luban/Luban.ClientServer/Luban.ClientServer.dll # 工具库
CONF_ROOT=${WORKSPACE}/Luban/DesignerConfigs # 配置目录

dotnet ${GEN_CLIENT} -j cfg --\ # 本地前后端一体, 就删除 -h ${LUBAN_SERVER_IP}
 -d ${CONF_ROOT}/Defines/__root__.xml \ # __root__.xml 定义文件的路径
 --input_data_dir ${CONF_ROOT}/Datas \ # 配置数据根目录(Datas)的路径
 --output_code_dir ${WORKSPACE}/Assets/Gen \ # 生成的代码文件的路径
 --output_data_dir ${WORKSPACE}/GenerateDatas/json \ # 导出的数据文件的路径
 --gen_types code_cs_unity_json,data_json \ # 生成代码,生成数据
 -s all

gen_code_json.bat

set WORKSPACE=..

set GEN_CLIENT=%WORKSPACE%\Luban\Luban.ClientServer\Luban.ClientServer.exe
set CONF_ROOT=%WORKSPACE%\Luban\DesignerConfigs

%GEN_CLIENT% -j cfg --^

  1. 加载配置

只需一行代码既可完成所有配置表的加载工具

csharp var tables = new cfg.Tables(file => JSON.Parse(File.ReadAllText(gameConfDir + "/" + file + ".json") ));

  1. 使用加载后的配置表

cfg.Tables 里包含所有配置表的一个实例字段。加载完 cfg.Tables 后，只需要用 tables.<表名> 就能获得那个表实例，接着可以做各种操作。例如我们要获取id = 10000 的那个道具。代码如下

csharp cfg.item.Item itemInfo = tables.TbItem.Get(10000); Console.WriteLine("id:{0} name:{1} desc:{2}", itemInfo.Id, itemInfo.Name, itemInfo.Desc);

可能你会注意到，item.xml 里定义 Item 时，字段名 id,name,desc的首字母被大写了。这是因为工具会根据输出的语言，自动作相应代码风格的字段名转换，也即 boo_bar 会被转换为 BooBar 这样的名字。这也是为什么推荐 配置字段定义时统一使用 xx_yy_zz 的风格。

至此完成 配置加载与使用!

配置详解

__root__.xml

自定义根文件, 不太需要去修改

__enums__.xlsx

自定义的枚举

__beans__.xlsx

自定义的结构

__beans__.xlsx 格式
full_name       *fields         ##var
                name    type    ##var
DataTest        test    string
                test2   string
----------------------------
excel 填写
VirtualCurrencyPrices   ##var
(list#sep=|),DataTest   ##type
虚拟货币
1,1|2,2
----------------------------
json 输出
{
    "test": "1,1",
    "test2": "2,2"
}

full_name 全名(包含模块和名字)

parent 父类(名字可以是全名，或者不包含命名空间，因为默认跟子类相同的命名空间)

alias 别名

sep 分割符

comment 注释

tags

*fields name 字段名 type 类型 group 分组 comment 注释 tags

__tables__.xlsx

自定义的索引表

full_name 全名(包含模块和名字)

goods.TbGoods // good(代码生成文件夹).TbGoods(模块名)

value_type 记录类名

Goods //就是类名

define_from_file 从excel读取定义 false时取已有定义，true为从excel标题头和属性栏读取定义

TRUE // TRUE才可以被命令工具读取

input 文件列表 可以多个，以逗号','分隔

goods/goods.xlsx // excel文档所在位置

index 表id字段 为空的话自动取value_type中第一个字段,多主键联合索引为key1+key2,多主键独立索引为"key1,key2"

id,name // 定义搜索key字段

mode 模式 取值one|map|list，为空自动为map

group 分组 取值c|s|e，可以有多个，以逗号','分隔。空则表示属于所有分组

comment 注释

patch_input 分支文件列表 用于本地化。可以多个，以逗号分隔

tags

output 输出文件名 默认为 _.``

options 额外选项 parser_mode=column或stream，默认为column

table 定义

定义一个表。 格式如下：

<table name="TbTableName" value="value_type" define_from_file="0" mode="map" input="file1,file2,..." group="group1,group2,..."/>
  • name 表名。无限制。但推荐用TbXxxx的格式。
  • value 表记录的结构类型名。
  • define_from_file 是否从文件中读取表记录value的定义。 可选。默认为false
  • mode 表模式。可选，默认为map。目前支持三种模式的表。
  • map 普通key-value表;
  • list 列表形式的表。但它支持0到多个索引，并且支持多主键联合索引及多主键独立索引。
  • singleton 为单例表，适合用于定义一些全局信息。
  • input 表数据源。可以有多个，以','分割。每个数据源可以是文件，目录，xlsx中的某个单元薄，json中的某个字段，等等。
  • group 表分组。 可选。值可以多个，以','分割，每个值必须为root.xml中定义的group中的一个。 如不填，则此表属于 root.xml中定义的group中default=1的分组。

数据结构

字段风格

item.xml 里定义 Item 时，字段名 id,name,desc的首字母被大写了。这是因为工具会根据输出的语言，自动作相应代码风格的字段名转换，也即 boo_bar 会被转换为 BooBar 这样的名字。 这也是为什么推荐 配置字段定义时统一使用 xx_yy_zz 的风格。

流式填写

(list#sep=;),Nodes#sep=,
itemId,1001,1;tableId,2001,2

list,bean 的水平展开多级子标题

##var items
##type list,Item
##var 0 1
##var item_id num desc item_id num desc
1 10 desc1 2 12 desc2
